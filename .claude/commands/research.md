# リサーチタスク

引数: `$ARGUMENTS`

## 指示

引数が GitHub issue の URL の場合（例: `https://github.com/owner/repo/issues/123`）:
1. GitHub MCP を使用して issue の内容を取得
2. issue のタイトル、本文、コメント、ラベル、マイルストーンを理解
3. 関連するコンテキストや依存関係を分析
4. 調査結果を日本語で要約

引数がその他の文字列の場合:
1. プロジェクト内で関連するコード、ファイル、ドキュメントを検索
2. 必要に応じて Explore エージェントを使用してコードベースを調査
3. 見つかった情報を分析
4. 調査結果を日本語で要約

## 出力形式

調査結果は以下の形式で報告してください:

- **概要**: 調査内容の簡潔な説明
- **詳細**: 発見した重要な情報
- **関連ファイル/コード**: 該当する場合、ファイルパスと行番号
- **推奨事項**: 次のステップや注意点（該当する場合）

すべての出力は日本語で行ってください。
